# MVP 범위 및 단계 게이트

## 한 줄 요약
P0(코어)→P1(리텐션)→P2(확장) 단계별 게이트 구조. 각 단계의 통과 지표를 못 넘기면 다음 단계에 착수하지 않는다 — 기능 과대 확장이 GDD 13장이 지목한 주요 리스크이기 때문.

## 핵심 타입 / 진입점
- 아직 코드 없음. 개발 착수 순서를 정할 때 이 문서를 기준으로 판단한다.

## 단계별 범위

| 단계 | 포함 기능 | 완료 판단 |
|---|---|---|
| **P0 코어 프로토타입** | 단순 낚시, 3종 요리, 소형 영지 배치, 코인 경제, 2인 협동 사냥, 기본 음성·음소거 | 15~30분 루프가 끊김 없이 순환하고 2인이 동일 목표·보상을 공유 |
| **P0 소셜 MVP** | 친구/룸 코드 입장, 오프라인 영지 읽기 방문, 관심 등록, 프리셋 리액션·사진 카드, 공개 범위 | 주인 부재 상태에서도 방문·기록·알림이 안정적으로 동작 |
| **P1 리텐션** | 레시피 숙련·빠른 조리, 주간 목표, 만찬, 날씨/시간 스폰, 숨은 수집물, 추천 영지 | 반복 플레이가 수집·꾸미기·방문으로 분산 |
| **P2 확장** | 공동 시설, 파티 무전, 고급 포탈 연출, 자유 필기/물리 방명록, 유료 테마 게스트 접근 | 모더레이션·성능·운영 비용 검증 후 단계 적용 |

## 범위 제외 (착수 금지)
UGC 월드 제작, 크리에이터 경제, 자동 생산/방치 시설, 대규모 오픈월드, 전투 중심 스킬 트리 — **코어 소셜 영지 루프 검증 전에는 착수하지 않는다.** 이 목록의 기능을 구현 요청받으면 이 문서를 근거로 확인을 요청할 것.

## 프로토타입 검증 지표 (P0 통과 기준)

시장 예측치가 아니라 **내부 프로토타입 통과 기준** — 테스트 표본에 따라 조정 가능.

| 검증 항목 | 초기 통과 기준 |
|---|---|
| 초기 이해도 | 80% 이상이 5분 내 첫 물고기 포획 |
| 루프 연결성 | 70% 이상이 첫 세션 20분 내 첫 오브젝트 배치 |
| 소셜 전환 | 첫 3회 세션 내 초대/방문 40% 이상 |
| 협동 품질 | 파티 유지율 60% 이상 / 진행도 불일치 1% 미만 |
| 방문 가치 | 방문객 30% 이상이 흔적(리액션·관심 등록) 남김 |
| 경제 균형 | 단일 활동이 전체 수익 60% 이상 독점하지 않음 |

## 시스템 간 관계
- P0 코어 = [[resource-loop]](낚시·요리·사냥 최소판) + [[estate-system]](소형 배치) + [[economy]](코인) + [[social-cooperation]](2인 협동·Shared Ledger 최소판) + [[room-architecture]](기본 Room 구조).
- P2의 "파티 무전"은 [[tech-stack-decisions]]의 Photon Voice 2 채널 기능으로 구현 예정 — P0에서는 근접 음성만.
- "자유 필기/물리 방명록"이 P2로 밀린 이유는 [[estate-system]]의 방명록 MVP 제약(프리셋 스탬프·사진 카드만)과 동일한 근거(저장 비용·모더레이션 부담).

## 기획 의도 / 역사적 맥락
- 게이트 구조를 명시한 건 GDD 13장 리스크 "기능 과대 확장 — 물리 방명록·포탈·UGC가 코어 개발을 잠식"에 대한 직접 대응. 각 단계의 통과 지표 미달 시 다음 단계 보류가 원칙.
- 협동 유도 개발 순서(방문 보너스 → 협동 사냥 → 공용 시설)는 [[social-cooperation]] 참조.

## 수정 시 주의
- 새 기능 요청이 들어오면 먼저 이 문서의 단계표에서 위치를 확인하고, P0 진행 중에 P1/P2 기능을 앞당기려면 사용자에게 게이트 원칙을 상기시킬 것.
- 검증 지표 수치를 바꾸면 GDD 12장과의 정합성을 함께 확인.
