# 영지 시스템

## 한 줄 요약
개인 소유 캠핑 영지. 제작이 아닌 구매로 오브젝트를 채우고, 무한 확장을 금지한 채 소형→상한 구조로 완성감을 확보한다. 방문객은 기본 읽기 전용.

## 핵심 타입 / 진입점
- 아직 코드 없음. Room 구조는 [[room-architecture]] 참조 (`roomName = estate_{userID}`).

## 핵심 규칙

| 항목 | 규칙 |
|---|---|
| 규모 | 시작은 소형(오브젝트 10~20개), 레벨/재화로 단계적 확장, **무한 확장 금지** — 명확한 최종 상한 존재 |
| 획득 방식 | 오브젝트는 **제작이 아닌 구매**로 획득 (크래프팅 시스템 자체를 배제해 단순화) |
| 방식 | 자동 생산/방치형 시설 **배제** — VR은 세션 기반 매체라 접속 중 직접 활동이 핵심이어야 함 |
| 방문 권한 | 방문객은 기본 읽기 전용. 주인이 명시적으로 허용한 공동 편집자만 오브젝트 이동 가능 |
| 성능 상한 | 오브젝트별 렌더링·물리 비용을 가중치로 환산한 "캠프 수용량" 적용. 유저에게는 기술 수치가 아니라 **남은 배치 가능량**으로 표시 |

## 오브젝트 카테고리 (초기 볼륨)

6개 카테고리 × 8~10개 = 약 50~60개, 등급은 일반/희귀/에픽 3단계.

| 카테고리 | 예시 | 등급 구성 |
|---|---|---|
| 쉘터 | 텐트, 타프 | 일반~희귀 |
| 조명 | 랜턴, 화롯불 | 일반~희귀 (야간 연출 핵심) |
| 가구 | 캠핑 의자, 테이블 | 일반 위주 |
| 장식/소품 | 화분, 소품 | 일반~에픽 |
| 바닥재/지형 | 데크, 잔디 패치 | 일반 |
| 특수 오브젝트 | 사운드/이펙트 시즌 한정템 | 에픽 (이벤트·협동 보상 전용) |

에픽 등급은 시즌 이벤트 및 커뮤니티 목표 완성 보상으로만 획득 — 상점에서 코인으로 직접 구매 불가.

## 방문 루프

발견(친구목록/최근 유저/관심 영지) → 방문(오프라인이어도 읽기 전용 입장, 하루 첫 3개 고유 영지까지 소량 보너스) → 상호작용(사진·스탬프·리액션·만찬·관심등록) → 재방문(업데이트된 배치·계절 연출 확인).

## 시스템 간 관계
- "구매 방식으로 단순화"는 [[resource-loop]]의 요리/사냥과 다른 경제 소스다 — 영지는 재화 **싱크**, 낚시/요리/사냥은 재화 **소스**. 이 둘을 헷갈려서 영지에 생산 기능을 넣으면 설계 위반.
- 방문 상태(읽기 전용 vs 공동 편집자)는 [[room-architecture]]의 "영지 권한: 배치 데이터는 서버 권한으로 저장, 주인·공동 편집자만 수정 가능"과 직결 — 클라이언트가 임의로 편집 권한을 판단하면 안 되고 서버가 최종 판정해야 함.
- 캠프 수용량(성능 상한)은 GDD 13장 리스크 "배치 성능 저하"의 직접 대응책. 오브젝트 추가 시 반드시 가중치를 매겨야 함 — 가중치 없는 오브젝트를 그냥 추가하면 이 안전장치가 무력화된다.

## 기획 의도 / 역사적 맥락
- 로스트아크 영지 시스템이 직접적 벤치마크지만, "자동 생산/방치형 시설(연구소식)"은 명시적으로 배제 — VR 세션 특성상 접속 중 직접 활동이 핵심이어야 한다는 판단 때문. 로스트아크를 그대로 가져오지 않은 지점이므로 유사 기능 제안 시 주의.
- 크래프팅 대신 구매 방식을 택한 건 시스템 단순화 목적 — "재료 조합 UI"는 요리 시스템에만 존재하고 영지 오브젝트에는 없다.

## 숨은 규칙 / 암묵지
- 배치 UX는 자유 배치와 스냅 배치를 **즉시 전환** 가능해야 하며, 복제·되돌리기·충돌 미리보기·원거리 집기 보조가 필요 (검증 지표 12장 "편집 피로" 항목과 연결).
- 자유 입력 텍스트/자필 3D 방명록은 MVP 범위 아님 — 프리셋 스탬프와 사진 카드로 제한 (저장 비용·부적절 콘텐츠 대응 부담 때문). 이 제약은 운영 도구가 준비된 이후에만 해제 대상.

## 수정 시 주의
- 캠프 수용량 가중치 테이블을 건드리면 [[social-cooperation]]의 공동 프로젝트(마을 공용 시설) 배치 여유량에도 영향을 줄 수 있음.
- 영지 배치 영구 저장 방식은 [[tech-stack-decisions]]에서 아직 미정 — 이 시스템의 실제 서버 저장 구현은 백엔드 확정 후 착수.
